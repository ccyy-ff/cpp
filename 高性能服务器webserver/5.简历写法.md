 **基于Linux平台的轻量级应用服务器 ** 

**项目描述**  
设计并实现了一个基于 Linux 平台的轻量级 HTTP 服务器，采用多 Reactor 多线程高并发模型，通过 epoll 提供高效的 I/O 复用。结合自动增长缓冲区、定时器和异步日志等技术，实现了高性能和稳定运行的目标。

**主要工作**

1. **内存优化**：设计了内存池和 LFU 缓存，减少内存碎片，提升内存使用效率。
2. **高效事件处理**：利用 epoll 多路复用机制，高效监听和处理客户端连接及数据传输事件。
3. **高并发模型**：基于 Reactor 模型，实现 One Loop per Thread，支持多客户端并发连接。
4. **动态缓冲区**：实现自动增长缓冲区，动态调整大小以适配不同请求，优化内存分配。
5. **连接管理**：使用小根堆实现高效定时器，管理连接超时时间，防止长期空闲连接浪费资源。
6. **异步日志**：设计异步日志模块，基于单例模式和阻塞队列，实现高效日志写入，避免同步写入的性能开销。

**项目难点**

1. **高并发设计**：需要设计高效的事件驱动模型，并优化 I/O 处理效率。
2. **负载均衡**：实现 One Loop per Thread 模型，确保每个连接在独立线程中高效运行。
3. **线程安全**：在多线程环境下，确保数据的同步与互斥，避免因竞争条件导致的错误。

**个人收获**

1. 深入理解了 Linux 网络编程，熟练掌握了 socket、epoll 和 TCP/IP 通信机制。
2. 学会了多 Reactor 模型的实现，提升了对高并发编程的理解和实践能力。
3. 掌握了动态缓冲区、定时器和异步日志等优化技术，提高了服务器性能和可靠性。
