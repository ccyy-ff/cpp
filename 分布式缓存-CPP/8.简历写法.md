# 8. 简历写法

## 项目：分布式缓存系统 KCache
### 项目描述
基于 C++17 实现的高性能分布式缓存系统，采用一致性哈希算法实现数据分片和负载均衡，支持多节点集群部署和动态扩缩容，实现了在分布式环境下的高效数据共享和访问。

### 个人工作
1. 实现了线程安全的LRU缓存淘汰算法；设计了基于字节大小的内存管理机制，支持自定义淘汰回调函数，优化内存使用效率
2. **实现了自适应一致性哈希算法**，支持虚拟节点动态调整，实时监控节点负载分布，动态调整虚拟节点数量确保数据均匀分布 
3. 采用读写锁（shared_mutex）实现高并发访问，支持多读单写，原子操作保证统计数据的线程安全性；**采用 SingleFlight 模式防止缓存击穿**，避免相同 key 的重复请求对后端服务造成压力
4. **基于 etcd 实现服务注册发现模块**，支持租约机制和自动续约，确保节点故障时的自动清理 
5. 基于 **gRPC** 实现了节点间通信协议，支持 Get/Set/Delete 操作
6. **实现了节点间数据同步机制**，本地写操作自动同步到对应的远程节点，保证分布式环境下的数据一致性
7. **实现了 HTTP 网关服务**，提供 RESTful API 接口，支持一致性哈希路由和轮询负载均衡策略

### 个人收获
+ 深入理解了分布式系统设计原则和最佳实践，特别是在数据分片、负载均衡和故障恢复方面的实现细节
+ 掌握了高并发编程技术，包括细粒度锁设计、原子操作和读写锁优化策略
+ 提升了 C++17 在系统级编程中的应用能力，特别是在多线程编程和网络编程方面
+ 学习了缓存淘汰策略的实现原理，以及在实际分布式场景中的应用和优化方法
+ 掌握了基于 etcd 的分布式协调技术，包括服务注册、健康检查和配置管理
+ 增强了对系统性能分析和优化的能力，能够通过性能监控识别瓶颈并进行针对性改进
