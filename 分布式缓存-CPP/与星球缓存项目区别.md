# 与星球缓存项目区别

[GitHub - youngyangyang04/KamaCache-CPP: 【代码随想录知识星球】项目分享-分布式缓存项目](https://github.com/youngyangyang04/KamaCache-CPP)

## 与 Go 版本分布式缓存的区别
本项目是代码随想录知识星球中[分布式缓存 Go 版本](https://wx.zsxq.com/group/88511825151142/topic/4845552222425428)的 C++实现，代码实现参考了 [GeeCache](https://github.com/geektutu/7days-golang/tree/master/gee-cache)和 [KamaCache-Go](https://github.com/youngyangyang04/KamaCache-Go)。

1. <font style="background-color:#FBDE28;">C++实现中没有完整复刻 Go 版本中的所有功能，不过核心功能都已经实现</font>
2. <font style="background-color:#FBDE28;">完善了缓存节点实例，添加了 HTTP 网关，可以让外部通过 restful api 来访问服务</font>
3. <font style="background-color:#FBDE28;">并且使用了 Dockerfile 一键构建镜像，Docker Compose 来管理容器集群</font>

这个项目更多的是想从 C++的方向来实现一下分布式缓存，看看 Go 中的实现在 C++中可以如何写，好让星球的 Cpper 们也能够一起讨论学习。项目总体逻辑和 Go 版本是一样的，因此这也不是一个新项目，文档更多是对于代码的理解与思考。

## 与 C++ 缓存系统项目的区别
星球中已有的 C++缓存系统项目实现了多种缓存替换策略，而最大的区别是** “C++缓存系统”更关注于本地，或者说单节点的缓存；而本项目更多关注于多个缓存节点之间的关系与通信**。

如果有时间深入学习，我觉得将二者融合起来也是个不错的 idea。

---

当然，我认为项目虽然比较简单，但是也可以去添加许多功能。比如说：

1. 增加不同的缓存算法；
2. 对于网关，为了避免单个控制层在流量暴增时宕机，可以使用 **多实例网关 + etcd 作协同（Leader Elected）**来增强网关的高可用性；
3. 实现缓存与数据的一致性；
4. ......
