# 3. SingleFlight 机制

## <font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">缓存三大问题：击穿、雪崩、穿透</font>

* **缓存击穿**：<font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">某个热点key在缓存过期瞬间，同时有大量请求访问这个key，导致所有请求都落到数据库上，造成数据库瞬时压力过大；</font>
* **<font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">缓存雪崩</font>**<font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">：大量缓存key同时失效，导致所有请求都落到数据库上，引起数据库压力激增甚至崩溃；</font>
* **<font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">缓存穿透</font>**<font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">：查询不存在的数据，导致每次请求都绕过缓存直接访问数据库。</font>

## 什么是 SingleFlight

SingleFlight 是一种用于合并并发请求的设计模式/机制，主要用于防止重复的昂贵操作（如数据库查询、HTTP请求等）被不必要地多次执行。

<font style="color:rgba(0, 0, 0, 0.85);">Singleflight 的核心思想是</font>**<font style="color:rgb(0, 0, 0) !important;">合并重复请求</font>**<font style="color:rgba(0, 0, 0, 0.85);">：当多个并发请求同时访问同一个资源（如缓存）时，只让第一个请求执行实际操作，其他请求等待该操作完成后直接复用结果。</font>

在分布式缓存这个项目中使用 SingleFlight 就是去\*\*<font style="background-color:#FBDE28;">合并并发请求，只允许一个请求访问数据源，防止缓存击穿</font>\*\*。

## 实现

```cpp
class SingleFlight {
    using Result = std::optional<ByteView>;
    using Func = std::function<Result()>;

public:
    Result Do(const std::string& key, Func func) {
        std::unique_lock<std::mutex> glock(mtx_);

        // 首先进行请求合并，检查是否有进行中的调用
        if (map_.find(key) != map_.end()) {
            auto existing_call = map_[key];
            glock.unlock();  // 释放组锁，避免阻塞其他键的处理

            // 直接等待 future 的结果
            auto result = existing_call->fut.get();
            return result;
        }

        // 创建新的调用对象
        auto new_call = std::make_shared<Call>();
        map_[key] = new_call;
        glock.unlock();

        // 执行用户函数并设置 promise
        Result val = func();
        new_call->prom.set_value(val);

        // 确保从映射中删除条目
        std::lock_guard<std::mutex> lock(mtx_);
        map_.erase(key);

        return val;
    }

private:
    struct Call {
        std::promise<Result> prom;
        std::shared_future<Result> fut = prom.get_future().share();
    };

    std::mutex mtx_;
    std::unordered_map<std::string, std::shared_ptr<Call>> map_;
};
```

> 对于 `Call` 结构体：
>
> * <code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">prom</font></code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">：</font><code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">std::promise<Result></font></code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">，用于设置异步操作的结果</font>
> * <code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">fut</font></code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">：</font><code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">std::shared_future<Result></font></code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">，共享的 future 对象，允许多个线程等待结果</font>
> * <font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">通过 </font><code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);">prom.get_future().share()</font></code><font style="color:rgba(0, 0, 0, 0.9);background-color:rgb(252, 252, 252);"> 创建共享的 future，使得多个线程可以安全等待。</font>

其工作流程可类比如下：

```latex
Thread 1: Get("user:123") → 创建 Call → 执行 func() → 设置结果 → 清理
Thread 2: Get("user:123") → 发现存在 Call → 等待结果
Thread 3: Get("user:123") → 发现存在 Call → 等待结果
Thread 4: Get("user:456") → 创建新 Call → 独立执行
```

简单来说，SingleFlight 将"多个相同请求"转换为"一次执行+多次共享"。
