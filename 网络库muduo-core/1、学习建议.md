## <font style="color:rgb(31, 35, 40);">并发模型</font>
![68747470733a2f2f63646e2e6e6c61726b2e636f6d2f79757175652f302f323032322f706e672f32363735323037382f313637303835333133343532382d63383864323766322d313061322d343664332d623330382d3438663736333261326630392e706e673f782d6f73732d70726f636573733d696d616765253246726573697a65253243775f3933372532436c696d69745f30](./img/Md1SP9J6OptwZLs0/68747470733a2f2f63646e2e6e6c61726b2e636f6d2f79757175652f302f323032322f706e672f32363735323037382f3136-653062.png)

<font style="color:rgb(31, 35, 40);">项目采用主从 多Reactor多线程 模型，MainReactor 只负责监听派发新连接，在 MainReactor 中通过 Acceptor 接收新连接并通过设计好的轮询算法派发给 SubReactor，SubReactor 负责此连接的读写事件。</font>

<font style="color:rgb(31, 35, 40);">调用 TcpServer 的 start 函数后，会内部创建线程池。每个线程独立的运行一个事件循环，即 SubReactor。MainReactor 从线程池中轮询获取 SubReactor 并派发给它新连接，处理读写事件的 SubReactor 个数一般和 CPU 核心数相等。使用主从 Reactor 模型有诸多优点：</font>

1. <font style="color:rgb(31, 35, 40);">响应快，不必为单个同步事件所阻塞，虽然 Reactor 本身依然是同步的；</font>
2. <font style="color:rgb(31, 35, 40);">可以最大程度避免复杂的多线程及同步问题，并且避免多线程/进程的切换；</font>
3. <font style="color:rgb(31, 35, 40);">扩展性好，可以方便通过增加 Reactor 实例个数充分利用 CPU 资源；</font>
4. <font style="color:rgb(31, 35, 40);">复用性好，Reactor 模型本身与具体事件处理逻辑无关，具有很高的复用性；</font>

## <font style="color:rgb(31, 35, 40);">编译指令</font>

<font style="color:rgb(31, 35, 40);">进入到muduo-core文件</font>

<font style="color:rgb(31, 35, 40);background-color:rgb(246, 248, 250);">cd muduo-core</font>

<font style="color:rgb(31, 35, 40);">创建build文件夹，并且进入build文件:</font>

<font style="color:rgb(31, 35, 40);background-color:rgb(246, 248, 250);">mkdir build && cd build</font>

<font style="color:rgb(31, 35, 40);">然后生成可执行程序文件：</font>

<font style="color:rgb(31, 35, 40);background-color:rgb(246, 248, 250);">cd example  &&  ./testserver</font>
