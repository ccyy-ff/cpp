# 0.学习建议

## 希望你有如下能力
1. 理解http通信过程
2. 掌握Linux下的网络编程
3. 熟悉C++11，如可变参数的使用，智能指针的使用
4. 熟悉Linux下的线程同步机制
5. 学习能力，因为会涉及到一些第三方库的学习
6. 理解多路复用的使用机制
7. 熟悉线程池
8. gdb调试的能力

## 需要学习的第三方库
1. Jsoncpp,用于对数据进行序列化的一个第三方库
    1. 要求了解json数据格式的数据结构，序列化反序列化方式
    2. 这一点直接搜索即可，很多博客，数据结构和序列化反序列化方式也很简单，了解用法即可
2. 了解libevent网络库的原理，http服务器的通信流程和使用方式，最好是能够挑选事件循环，库对几种多路复用机制是怎么应用的一些方面去进行深入的了解一下，对于该项目来说只需要会用即可，但是面试很大几率会问你libevent相关内容。
    1. [这是官方写的用来熟悉libevent的文档](https://github.com/huyubing/books-pdf/blob/master/libevent%E5%8F%82%E8%80%83%E6%89%8B%E5%86%8C(%E4%B8%AD%E6%96%87%E7%89%88).pdf)
    2. [这是传音博客的教程](https://aceld.gitbooks.io/libevent/content/)
    3. 还有一门课叫《51CTO夏曹俊-C++跨平台Libevent高并发网络编程》，这个可以去搜一搜，是付费的，自己找资源吧不好给链接。建议可以看看这个，课程的时间不长，看完这个就可以用libevent有一个ok的了解上手写代码了。前面两个也ok。
    4. 这边建议的话，时间允许的情况下，libevent源码是很好读的，对于学生来说了解一个比较常用的网络库实现也是比较加分的，它的特性是轻量级，高性能的事件通知库，那么就可以去了解下为什么高性能，他的事件循环怎么做的，对我们学的epoll之类的多路复用机制是怎么应用的。能够跟面试官说道说道这个我觉得才是加分的地方，项目只是一个引子。
3. bundle库，用于压缩
    1. 这里的话就熟悉用法，然后知道几种压缩格式和各自的特点就好，面试可能会问选的什么压缩格式

## 建议
_整个项目大致2000行代码_

1. 基础OK的同学，把以上提到的第三方库了解完，大概7小时以内就ok，jsoncpp和bundle2小时之内就可以学完，libevent可以学久一点，多了解了解。然后后面的话项目写完了有时间再去看看源码整理些笔记。**学完以上预备知识，整个项目每天花7小时，大致20天即可完成**。
2. 基础不OK需要学习我上述所说的"希望你有如下能力的同学",**每天7小时，多花个一周即可**。主要可能会出现一些bug自己不会排查，需要多练习掌握gdb的使用。
3. 后期项目写完多花时间在libevent的学习上
4. 学习以上知识时建议自己手搓代码而不只是看别人写的文章，比如自己先简单使用libevent实现一个http通信，自己完成一下json数据的序列化反序列化。
5. 后续建议大家自行扩展项目，最后项目写完给了一些可扩展的点，大家也可以自行探索，可以侧重在加强性能而不是功能方面，功能方面能用就行。
6. 客户端（在备份中）可以实现也可不实现，因为和服务端逻辑差不多，也并不复杂，咱主要还是关注服务端。且客户端的libevent环境配置比较麻烦，可能会遇到问题，具体看同学的时间安排。如果配置完了，同学也是第一次配置的话可以写篇博客记录这个过程总结一些经验。
7. 存储的前端设计了html和js代码等前端代码，包括后端Service.hpp中的几个用来解析组织html的函数，不会的同学可以直接复制，这个不是重点
